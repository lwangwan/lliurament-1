Diagramas de casos de uso textuales de los casos de uso que tengan includes y excludes
CU2: Veure detall de la sèrie
Breve descripción: Este caso de uso permite al usuario ver: el título de la serie, una breve descripción,
los artistas que participan (director, actores y productora) y sus datos; el número de temporadas y el número de episodios
por temporada; el título del episodio, la duración, el idioma original, una breve descripción y la
fecha en que se emitió por primera vez.


Actores: cliente registrado
Precondiciones: El cliente ha abierto el catálogo.

Flujo de eventos:El caso de uso se inicia cuando el cliente selecciona una serie.
Flujo básico:
1.	El cliente selecciona una serie.
2.	El sistema muestra la ventana con la información de la serie.
	a)
	......

3.	El cliente lee los detalles de la serie.
4.	El cliente sale de la ventana y vuelve a la pantalla del catálogo. El caso de uso finaliza.

Flujo alternativo: 
4 a. El cliente registrado selecciona ver un episodio.
	1.	El sistema carga el episodio.
	2.	El cliente cierra la ventana de reproducción.
	3.	El sistema vuelve a la pantalla de detalle de la serie.

Postcondiciones: (Actualizar "Big Data") VS Ninguna




CU3: Veure catàleg de series
Breve descripción: Al entrar en la aplicación, el cliente dispone del catálogo de series disponibles.
Actores: cliente no registrado, cliente registrado
Precondiciones: Ninguna

Flujo de eventos: El caso de uso se inicia cuando el cliente entra en la aplicación.
Flujo básico:
1.	El cliente abre la aplicación.
2.	El sistema muestra la ventana con todas las series que dispone.
3.	El cliente ve el catálogo.
4.	El cliente selecciona salir de la aplicación.
5. 	El sistema cierra la aplicación y finaliza el caso de uso.

Flujo alternativo: CU2
4 a. El cliente selecciona ver detalles de una serie. 
Postcondiciones: Ninguna




CU6: Veure episodi
Breve descripción: Al entrar en los detalles de una serie, el cliente selecciona un capítulo.
Actor: cliente registrado 
Precondiciones: Haber seleccionado una serie
Flujo de eventos: El caso de uso se inicia cuando el cliente selecciona un episodio para verlo.

Flujo básico:
1.	El cliente selecciona el episodio de la serie que quiere ver.
2.	El sistema muestra una nueva ventana donde se carga el episodio.
3.	El cliente ve el capítulo.
4.	El cliente sale de la ventana.

5. 	El sistema muestra la opción para valorar el episodio.
6.	Incluir caso de uso "Valorar episodi (CU7)" y el caso de uso finaliza.


Flujo alternativo:
6.a	En el paso 6 del flujo básico, el cliente puede seleccionar salir y el 
	caso de uso finaliza.
Postcondiciones: Se guarda la información (de visualización y valoración).




CU7: Valorar episodi
Breve descripción: El cliente ha visualizado el episodio y se le muestra la opción de valorarlo.
Actor: cliente registrado 
Precondiciones: Haber visto el episodio

Flujo de eventos: El caso de uso se inicia cuando el cliente termina de ver un episodio.

Flujo básico:
1.	El cliente termina de ver el episodio
2.	El sistema muestra la opción de valorar el episodio:
	a) indica el identificador del usuario
	b) valoración del 1 al 5.
3.	El cliente valora el episodio.
4.	El sistema registra la valoración.
5.	El cliente selecciona salir de la aplicación.
6. 	El sistema cierra la aplicación y finaliza el caso de uso.


Postcondiciones: La valoración queda guardada en el sistema.
